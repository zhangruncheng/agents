# Claude Code 配置优先级说明

## 概述

Claude Code 采用层级化的配置管理系统，支持从企业级到项目级的多层配置覆盖机制。配置按照特定性原则进行优先级排序，越具体的配置优先级越高。

## CLAUDE.md 文件优先级

### 优先级顺序（从高到低）

1. **项目级别** `.claude/CLAUDE.md`
   - 位于项目根目录下的 `.claude/` 文件夹中
   - 针对特定项目的自定义指令
   - 完全覆盖用户级别的配置

2. **用户级别** `~/.claude/CLAUDE.md`
   - 位于用户主目录的 `.claude/` 文件夹中
   - 作为所有项目的默认配置基础
   - 当项目级别无配置时生效

### 覆盖规则

- 项目级别的 CLAUDE.md 会**完全替换**用户级别的配置，而非合并
- 如果项目中没有 CLAUDE.md 文件，则使用用户级别的配置

## Agents 和 Commands 优先级

### Agents 优先级顺序

1. **项目级别** `.claude/agents/`
   - 项目特定的智能体定义
   - 可以覆盖同名的用户级别智能体

2. **用户级别** `~/.claude/agents/`
   - 全局可用的智能体库
   - 作为默认智能体集合

### Commands 优先级顺序

1. **项目级别** `.claude/commands/`
   - 项目特定的自定义命令
   - 优先于用户级别的同名命令

2. **用户级别** `~/.claude/commands/`
   - 全局可用的命令库
   - 跨项目共享的通用命令

## 完整配置系统优先级

### 从高到低的完整优先级顺序

1. **企业管理策略** `managed-settings.json`
   - 最高优先级，不可覆盖
   - 确保企业安全和合规要求
   - 由系统管理员统一配置

2. **命令行参数**
   - 临时会话级别的覆盖
   - 仅在当前执行中生效
   - 用于临时调整配置

3. **本地项目设置** `.claude/settings.local.json`
   - 项目级别的个人配置
   - 通常不纳入版本控制
   - 适合个人开发偏好设置

4. **共享项目设置** `.claude/settings.json`
   - 项目级别的团队共享配置
   - 通常纳入版本控制
   - 确保团队配置一致性

5. **用户全局设置** `~/.claude/settings.json`
   - 用户级别的默认配置
   - 作为所有项目的基础设置
   - 最低优先级

## 配置策略建议

### 团队协作建议

- **共享配置**: 使用 `.claude/settings.json` 定义团队通用设置
- **个人配置**: 使用 `.claude/settings.local.json` 进行个人定制
- **版本控制**: 将 `settings.local.json` 添加到 `.gitignore`

### 项目配置建议

- **项目特定需求**: 在 `.claude/CLAUDE.md` 中定义项目特有指令
- **复用全局配置**: 利用用户级别配置作为基础模板
- **智能体管理**: 项目特定智能体放在 `.claude/agents/`

### 用户配置建议

- **全局默认**: 在 `~/.claude/CLAUDE.md` 中设置通用指令
- **智能体库**: 在 `~/.claude/agents/` 中维护常用智能体
- **命令库**: 在 `~/.claude/commands/` 中存储跨项目命令

## 核心原则

### 1. 就近原则
越接近项目的配置优先级越高，确保项目特定需求得到满足。

### 2. 特定性原则  
项目特定配置始终覆盖全局配置，支持精确的项目定制。

### 3. 安全优先原则
企业管理策略始终具有最高优先级，确保安全和合规要求。

### 4. 灵活性原则
支持多层次配置，既保证统一性又允许个性化定制。

## 实际应用示例

### 场景1：个人开发者
```
~/.claude/CLAUDE.md          # 个人偏好和通用指令
~/.claude/agents/           # 常用智能体库
项目A/.claude/CLAUDE.md      # 项目A特定指令
项目B/.claude/CLAUDE.md      # 项目B特定指令
```

### 场景2：团队协作
```
~/.claude/CLAUDE.md                    # 个人基础配置
项目/.claude/settings.json             # 团队共享配置
项目/.claude/settings.local.json       # 个人项目配置
项目/.claude/CLAUDE.md                 # 项目团队指令
```

### 场景3：企业环境
```
managed-settings.json                   # 企业策略（最高优先级）
~/.claude/settings.json                 # 用户基础配置
项目/.claude/settings.json              # 项目配置
项目/.claude/settings.local.json        # 个人项目配置
```

## 注意事项

1. **配置冲突**: 高优先级配置会完全覆盖低优先级的同名配置项
2. **文件缺失**: 如果某级别配置文件不存在，自动使用下一优先级的配置
3. **权限管理**: 企业管理策略可能限制某些配置的修改权限
4. **调试技巧**: 可以通过命令行参数临时覆盖配置进行测试
